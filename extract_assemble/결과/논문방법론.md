# 연구 방법

## 1. 연구 대상과 데이터

본 연구는 한국 근현대 문학비평 텍스트를 TEI(Text Encoding Initiative) 기반 XML로 자동 인코딩하는 방법을 제안하고 그 성능을 검증한다. 분석 대상은 2021년 발표된 두 편의 신인 평론 — 성현아의 「자본주의에 대처하는 우리의 자세」와 김주원의 「휴머니즘의 외부와 열림의 존재론」 — 이다. 두 텍스트는 PDF에서 평문으로 추출하였으며, 각각 약 16,000자 규모이다.

인코딩의 표적 구조는 협업 연구진이 설계한 도메인 특화 TEI 스키마(`korean-critique-schema.xsd`)로 규정된다. 이 스키마는 비평 텍스트에 특화된 일곱 가지 인라인 개체를 정의한다: 인물명(`persName`), 작품·문헌명(`title`), 기관·매체명(`orgName`), 비평 용어(`term`), 인용(`quote`), 해석·평가(`interp`), 연도·날짜(`date`). 각 개체는 통제 어휘(controlled vocabulary)로 제한된 속성을 가진다(예: `persName/@role`은 비평가·시인·번역가·외국 문인 등 열한 개 값으로 제한). 또한 본 연구 과정에서 스키마를 확장하여(v5) 본문 내 `date`의 정식 허용, 시점·기간을 구분하는 날짜 속성(`when`, `from`/`to`), 운문 구조(`lg`/`l`) 등을 반영하였다.

정량 평가를 위해 성현아 비평문 한 편을 연구자가 위 스키마에 따라 수작업으로 태깅하여 **정답(gold standard)** 을 구축하였다. 김주원 비평문은 정답을 구축하지 않은 독립 테스트 대상으로 두어, 동일 텍스트에 대한 과적합을 배제하였다.

## 2. 시스템 아키텍처: 추출-후-조립(Extract-then-Assemble)

대규모 언어모델(LLM)을 이용해 텍스트를 직접 TEI/XML로 **생성**시키는 직관적 접근은 두 가지 구조적 한계를 가진다. 첫째, 모델이 장문의 본문을 매 호출마다 재생산하므로 청크 경계에서 내용이 누락·변형될 수 있다(본문 일부 또는 단락 단위 소실). 둘째, 모델의 비결정적 출력이 스키마의 구조·어휘 제약을 빈번히 위반한다(태그 중첩, 식별자 중복, 허용값 외 속성 등). 본 연구의 사전 실험에서 단일 비평문 출력에 56건의 스키마 위반과 간헐적 분량 손실이 관찰되었다.

이러한 한계는 **"문서 구조를 생성하는 주체가 LLM"** 이라는 점에서 기인한다. 본 연구는 역할 분담을 재설계하여, LLM에게는 **의미 판단**(어떤 문자열이 어떤 개체인가)만을 맡기고, **문서 구조의 생성·검증·완결성 보장**은 결정론적 코드가 담당하게 하였다. 이를 추출-후-조립(extract-then-assemble) 아키텍처로 명명한다.

```
원문 ──▶ ① 구조화(코드) ──▶ ② 개체 라벨링(LLM) ──▶ ③ 전역 해소(LLM/코드)
                                                  ──▶ ④ 규칙 기반 보강(코드)
                                                  ──▶ ⑤ 스키마 준수 조립(코드) ──▶ 검증된 TEI
```

핵심 원리는 두 가지다. (가) **코드는 원문을 변형하지 않고 태그만 삽입**하므로 분량 손실이 원천적으로 불가능하다. (나) **코드는 스키마 규칙대로만 조립**하므로 구조·어휘 위반이 발생할 수 없다. LLM의 실수는 라벨의 정확도(정밀도·재현율)에만 영향을 미치며, 문서의 유효성과 완결성을 훼손하지 못한다. 본 연구는 LLM으로 Claude(Sonnet 4.6)를 사용하였다.

## 3. 처리 단계

### 3.1 텍스트 구조화

원문을 섹션(`div`)·단락(`p`)·문장(`s`)의 골격으로 분해한다. 이 단계는 LLM을 사용하지 않으며 원문을 그대로 보존하므로 분량이 100% 유지된다. 다만 PDF에서 추출한 평문은 인쇄 판형에 따른 강제 줄바꿈이 단어 중간을 절단하는 문제를 가진다. 이를 보정하기 위해 줄바꿈을 문장 경계로 오인하지 않고, 줄을 재결합한 뒤 종결 부호를 기준으로만 문장을 분리하였다.

### 3.2 개체 라벨링(LLM)

골격의 각 문장에 대해 LLM이 개체를 라벨링한다. 모델은 문장 텍스트를 변경하지 않고, 문장 내 부분 문자열(span)을 지목하여 태그 유형과 속성을 JSON으로 반환한다. 예:

```json
{"text": "최지인", "tag": "persName", "attrs": {"role": "poet"}}
```

문맥 손실을 줄이기 위해 같은 단락의 연속된 문장을 한 묶음으로 제시하고, 소수의 예시(few-shot)를 통해 태깅 밀도와 범위를 학습시켰다. LLM 출력은 곧 라벨 데이터이므로, 동일 라벨에 대해 조립 규칙만 변경하여 추가 호출 없이 문서를 재생성(재조립)할 수 있다. 이는 후속 단계의 정책 변경과 재현 실험을 경제적으로 만든다.

### 3.3 라벨 영속화와 재조립을 통한 재현성·경제성

추출-후-조립 구조에서 LLM 라벨링(비결정적·고비용)과 조립(결정론적·무비용)은 명확히 분리된다. 본 연구는 LLM이 산출한 라벨을 JSON으로 영속화(persist)하여, 이후 조립 단계를 라벨 재호출 없이 반복 수행할 수 있게 하였다. 라벨링이 한 번 수행되면 그 결과가 자동 저장되며, 후속 처리는 이 저장된 라벨을 입력으로 재조립(re-assembly)하는 방식으로 이루어진다. 이는 두 가지 방법론적 이점을 가진다.

첫째, **재현성**이다. 동일한 라벨은 항상 동일한 문서로 조립되므로, 조립 정책의 변경 효과를 LLM의 실행 간 변동(run-to-run variance)과 분리하여 관찰할 수 있다. 예컨대 날짜 표기 정책, 인물 역할 해소, 인용 보강, 운문 구조화 등 후처리 규칙의 영향을 동일 라벨 위에서 통제 비교(controlled comparison)할 수 있다. 이는 후처리 규칙의 효과 검증을 모델 출력의 우연한 변동으로부터 보호한다.

둘째, **경제성**이다. 조립 정책을 조정하는 일련의 실험이 새로운 LLM 호출 없이 수행되므로 비용과 시간이 크게 절감된다. 본 연구의 후처리 규칙(3.4~3.7)은 모두 영속화된 라벨을 재조립하는 방식으로 검증되었다.

영속화되는 라벨은 전역 해소·규칙 기반 보강 이전의 **원시 LLM 출력**으로, 후처리 정책과 독립적으로 보존된다. 따라서 후처리 정책을 교체·누적하더라도 동일한 의미 판단 위에서 비교가 성립한다.

### 3.4 전역 해소: 인물 역할

문장 단위 라벨링은 개별 인물의 일관된 역할을 파악하기 어렵다. 한 인물이 문장마다 다른 역할로 태깅되거나(예: 외국 이론가를 학자로), 정답의 표기 관행과 어긋날 수 있다. 이를 위해 문서 전체의 인물 목록을 LLM에 일괄 제시하여 **인물당 역할을 하나로 확정**하고 전역적으로 적용한다(외국 문인·이론가는 `foreigner`로 통일). 또한 정답의 관행에 따라 **역할 속성은 인물의 첫 등장에만 부여하고, 반복 언급은 참조(`ref`)로 연결**하여 식별자 중복과 속성 불일치를 동시에 해소한다.

### 3.5 규칙 기반 인용 보강

LLM은 짧은 인라인 인용(따옴표로 묶인 어구)을 빈번히 누락한다. 따옴표는 객관적 신호이므로, 문장 내 큰·작은따옴표 안의 어구를 결정론적으로 `quote`로 보강한다. 기존 라벨과 중첩되는 경우는 제외하며, 장르·유형 속성은 문서 내 기존 인용의 다수값을 따른다. 이 규칙 기반 보강은 비결정적 프롬프트 조정과 달리 과태깅 위험이 낮고 재현 가능하다.

### 3.6 스키마 준수 조립

조립기는 골격과 라벨을 받아 원문 위에 태그를 삽입하여 완결된 TEI 문서를 구성하고, 조립 시점에 스키마 제약을 강제한다. 구체적으로 (가) 식별자는 첫 등장에만 부여하고 반복은 참조로 처리하여 **고유성**을 보장하며, (나) 위치가 겹치는 라벨은 비중첩 원칙에 따라 선택하여 **태그 중첩**을 방지하고, (다) 역할·수준·유형 등 속성을 허용 어휘로 **정규화**(허용값 외는 제거)하며, (라) 메타데이터 헤더(`teiHeader`)와 분류체계(`taxonomy`)를 고정 구조로 완비한다. 조립 결과는 XSD에 대해 즉시 검증된다.

### 3.7 운문 구조 복원

비평 텍스트는 시를 행 단위로 인용하지만, 구조화 단계에서 줄바꿈이 제거되어 행 정보가 소실된다. 이를 복원하기 위해, 시 인용(`genre="poet"`)에 대해 인용 어구를 원문에서 공백 무시 정합으로 탐색하여 원래의 줄바꿈 위치를 회수하고, 각 행을 연(`lg`)·행(`l`) 구조로 재구성한다. 생략 표지(…)는 별도의 주석(`note`)으로 처리한다. 이로써 자동 인코딩이 수작업 정답의 운문 마크업 관행과 정합한다.

## 4. 평가 방법

### 4.1 지표

자동 태깅 결과를 정답과 비교하여 개체 유형별 정밀도(precision)·재현율(recall)·F1을 산출한다. 채점은 두 수준으로 수행한다. **느슨(lenient)** 채점은 태그 유형과 텍스트의 일치만을 보아 "개체를 식별하였는가"를 측정하며, **엄격(strict)** 채점은 속성(역할·유형·값 등)까지의 일치를 요구하여 "올바르게 분류하였는가"를 측정한다. 두 점수의 차이는 곧 속성 분류 오류율을 의미한다. 정답과 자동 출력은 각각 개체 다중집합(multiset)으로 변환되며, 다중집합 교집합으로 일치 개수를 센다. 비교 시 표기 차이(따옴표·낫표·각주 번호 등)는 정규화하여 마크업 관행의 차이가 점수를 왜곡하지 않도록 하였다.

해석(`interp`)과 같이 태깅 범위가 본질적으로 주관적인 개체는 정밀 일치 지표가 과소평가하는 한계가 있어, 정량 평가의 보조 지표로만 다룬다.

### 4.2 완결성과 유효성 검증

정확도와 별개로, 자동 출력이 원문을 빠짐없이 포함하는지를 **분량 커버리지**로 검증한다. 원문과 출력 본문을 문장·글자 단위로 대조하여 누락을 탐지한다. 또한 모든 출력은 대상 XSD에 대해 검증하여 **스키마 위반 건수**를 보고한다. 제안 아키텍처에서 두 지표는 설계상 각각 100%와 0건이 보장된다.

### 4.3 실험 설계

과적합을 배제하기 위해 정답이 구축된 비평문(성현아)으로 지표를 산출하고, 정답이 없는 비평문(김주원)은 유효성·완결성·태깅 분포만으로 일반화 가능성을 점검하였다. 이로써 특정 텍스트에 대한 우연한 적합을 통제하였다.
