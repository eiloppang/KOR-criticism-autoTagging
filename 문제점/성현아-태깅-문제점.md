# 성현아 비평문 태깅 출력 — 문제점 및 해결 방안

- **대상**: `비평문 아웃풋.xml` (성현아 「자본주의에 대처하는 우리의 자세 — 최백규와 최지인이 노동과 우울을 그리는 방식에 대하여」 자동 태깅 결과)
- **정리일**: 2026-06-15
- **목적**: 이 출력을 **성현아 정답(gold)의 초안**으로 다듬기 위한 문제 진단
- **종합 판단**: 태그 커버리지와 속성(role/level/type/value/ana/when/source)은 **우수**. 다만 **개체 동일성(xml:id) 관리**와 **잡음 제거**에서 손볼 곳이 있음.

## P1. 같은 인물이 여러 id로 파편화 + 중복 id + 음역 불일치 ★

### 증상
- **파편화**: 최백규 한 명이 `p-choibaekgyu` ~ `p-choibaekgyu13` (id 13개), 최지인 `p-choijiin`~`p-choijiin9`
- **음역 3종**: 양경언이 `p-yangkyeongeon` / `p-yangkyungun` / `p-yanggyeongeon` 세 가지 라틴 음역으로 갈림 (텍스트는 모두 "양경언")
- **중복 id**: 본문 `p-markfisher`(role="foreigner scholar")와 각주 `p-markfisher`(role="scholar foreigner")가 **동일 id** → XML ID 고유성 위반. `p-choibaekgyu`·`p-choijiin`도 본문/각주에서 중복

### 영향
- `개체명 데이터` 탭에서 최백규가 빈도 13이 아니라 **빈도 1짜리 13행**으로 흩어짐 → 통계·시각화·네트워크 전부 왜곡
- 중복 id 때문에 엄격 XML 파서가 파싱 거부 (현재는 `collect_ids=False`로 우회 중)

### 해결 방안
- **`_dedup_xml_ids`(이미 존재)를 적용** → (태그, 정규화 텍스트) 기준으로 통일.
  - "양경언" 텍스트가 동일하므로 음역 3종도 **자동으로 하나로 병합**됨
  - 최백규 13개 → 1개 canonical로 수렴
- 중복/파편 id 제거 후 **canonical id 하나로 통일**
- ⚠️ 주의: `role` 속성 표기 순서 불일치(`foreigner scholar` vs `scholar foreigner`)는 같은 집합이므로 **정렬해서 통일** 필요 (평가 엔진은 이미 집합으로 비교)
- **자동화**: 가능 (이 출력의 role은 `#id` 참조를 안 쓰므로 교차참조 깨짐 없음)

## P2. 잡음 태깅

### 증상
- **생략부호를 인용으로**: `<quote type="direct" genre="poet" source="장마철">(…)</quote>` — "(…)"는 시 인용 생략 표시일 뿐 인용문이 아님 (여러 곳)
- **웹 잡음**: 맨 끝 `<s>FacebookTwitterKakaoLine</s>` — PDF/웹에서 딸려온 공유버튼 텍스트

### 영향
- 정답·평가 데이터에 무의미 항목이 섞여 정밀도 왜곡

### 해결 방안
- 규칙 기반 제거:
  - `quote` 중 내용이 `(…)`, `(...)`, `…`, 공백뿐인 것 → 태그 해제 또는 문장 삭제
  - "FacebookTwitterKakaoLine" 같은 공유버튼/메타 텍스트 줄 삭제
- **자동화**: 가능 (정규식·화이트리스트)

---

## P3. title 괄호(「」) 일관성

### 증상
- `「비정규」`처럼 낫표 포함한 제목과 `컨베이어`처럼 낫표 없는 제목이 섞임

### 영향
- 같은 작품이 표기에 따라 다른 텍스트로 취급 → 매칭·집계 불일치

### 해결 방안
- **규칙 결정 필요**: title 텍스트에 낫표(「」)를 **포함할지/제외할지** 하나로 정함
  - 권장: **제외** (제목 텍스트만, 낫표는 마크업 외부) — 매칭·집계에 유리
- 결정 후 일괄 정규화
- **자동화**: 결정 후 가능

---

## P4. 섹션 head 누락 의심

### 증상
- 섹션 head가 "1. 분노 없이도" → "2. 아버지-아들…" → **"4. 벗기 위해 청소하는 시"** 로 이어짐. **3번 head가 안 보임**

### 영향
- 청크 병합 경계에서 섹션 제목이 누락됐을 가능성 (구조 손실)

### 해결 방안
- **원문(성현아.txt) 대조**로 3번 섹션이 실제로 존재하는지 확인
  - 원문에 있으면 → head 복원
  - 원문도 3이 없으면(저자 번호 매김 특성) → 문제 아님
- **자동화**: 불가 (사람이 원문 대조 — 단, 원문에서 "^3\." 패턴 탐색으로 보조 가능)

---

## P5. taxonomy↔role 미연계 · date 인라인 (스키마 관련)

### 증상
- 헤더 taxonomy는 `xml:id="cat-critic"`인데 본문은 `role="critic"` → `role`이 `#cat-critic`을 가리키지 않음 (느슨한 연결)
- `date` 인라인 태그는 현재 `korean-critique-schema.xsd`의 SentenceType에 미정의 → **XSD 검증 시 경고** (validator.py에 명시된 알려진 제약)

### 영향
- taxonomy 연결이 형식적 — 분석엔 지장 적음
- date 경고는 검증 결과에 계속 뜸

### 해결 방안
- taxonomy: 분석 목적이면 현행 유지 가능. 엄밀한 TEI면 `role` 값을 taxonomy id와 일치시키거나 `#` 참조로 변경 (설계 결정)
- date: **XSD는 수정 금지(협업자 설계)** 이므로, 경고를 **수용**하거나 별도 정책 합의 필요
- **자동화**: 설계 결정 사항이라 보류

---

## 해결 전략 (단계)

```
1차 (자동, 안전):   P1 id 통일 + P2 잡음 제거
                    → cleaned 초안 생성
2차 (결정 후 자동): P3 title 낫표 정책 적용
3차 (사람 대조):    P4 섹션 3 확인/복원
4차 (설계 결정):    P5 taxonomy/date 정책
5차 (사람 검수):    role/type/value 등 속성 정확성 최종 확인
                    → 성현아 gold(정답) 확정
```

**핵심**: 1차(P1·P2)만 돌려도 데이터 품질이 크게 개선되고, 이후 사람은 **속성 검수에만 집중**하면 됨.
